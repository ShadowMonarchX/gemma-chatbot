Strategy 1 — quantization (biggest single win)
Q
High impact
AWQ 4-bit quantization
Gemma 3 27B: BF16 = 54GB VRAM → AWQ = 14GB VRAM. Same g5.12xlarge now runs 2 model replicas instead of 1. Double the throughput, same cost. Quality drop: ~1–2% on benchmarks — imperceptible in production.

--quantization awq --dtype half in vLLM startup. Download AWQ weights: google/gemma-3-27b-it-awq from HuggingFace.
54GB → 14GB VRAM 2× more users same GPU

EC2 Spot instances
Spot = same hardware, 60–70% cheaper. Risk: AWS can reclaim with 2-min warning. Mitigate with: Spot + On-Demand mixed fleet (50/50). If Spot disappears, On-Demand handles traffic. Use Spot for stateless inference workers — they can die anytime since each request is independent.

Setup: ASG with mixed instances policy. OnDemandBaseCapacity: 1, rest Spot.

C
Medium impact
Semantic response caching
Cache LLM responses in Redis. For identical or near-identical prompts, return cached result — zero GPU cost. Exact cache hit: use prompt hash as key. Semantic cache: embed the prompt, find nearest cached response if cosine similarity > 0.95.

GPTCache library does this automatically. In production, 20–40% of requests are cache hits (FAQs, repeated queries).
20–40% requests free

Right-size: smaller model for simple tasks
Not every task needs 27B parameters. Route simple queries to a smaller, cheaper model:
• Simple FAQ / classification → Gemma 3 2B (10× cheaper, fits on T4)
• Complex reasoning / code / agents → Gemma 3 27B

Use a fast classifier (even regex or a tiny 0.5B model) to route requests. This is called model routing or LLM cascade.

Speed strategy — ordered by impact
1
Biggest win
Always stream tokens to user
User sees first word in ~100–300ms even if full response takes 5 seconds. This is the single highest-impact change for perceived speed — zero infra cost.

stream=True in your API call. Frontend renders tokens as they arrive via SSE. The user's brain registers this as "fast" even for long responses.
100ms felt latency zero infra cost
2
High impact
Speculative decoding (draft model)
Run a tiny 2B "draft" model alongside Gemma 27B. Draft model proposes 4–6 tokens at once. Big model verifies them in parallel. When draft is right (most of the time), you get 4–6 tokens for the cost of 1 LLM forward pass. Result: 2–3× faster token generation.

vLLM supports this: --speculative-model google/gemma-3-2b-it --num-speculative-tokens 5
2–3× faster generation
3
High impact
Parallel tool calls in agent loop
Instead of calling tools one-by-one (sequential: 3 tools × 500ms = 1.5s), call all independent tools simultaneously. LangGraph supports parallel tool execution natively.

Example: agent needs weather + stock price + news → fire all 3 MCP calls at once → collect results → single LLM reasoning step. 1.5s → 0.5s for that step.
3× faster agent tool phase
4
Medium impact
Chunked prefill + prefix caching
vLLM prefix caching: if multiple requests share the same system prompt (they always do), the KV-cache for that prefix is computed once and reused. 0ms prefill cost for the shared part.

Enable: --enable-prefix-caching in vLLM. For agents with a long system prompt, this can cut TTFT by 50–70%.
50–70% lower TTFT same system prompt reuse
5
Medium impact
Warm pool — eliminate cold starts
Gemma 3 27B takes 60–90s to load on a fresh EC2. With a warm pool, ASG pre-launches instances and loads the model before any traffic hits them. Users never see a cold start.

ASG warm pool config: MinSize: 1 always warm. Scale-out pulls from warm pool (ready in ~10s) instead of launching cold (90s).
90s → 10s scale-out

Tier 1 — Cache + small model (free / ultra-cheap)
Redis exact cache → semantic cache (GPTCache) → Gemma 2B on g4dn.xlarge ($0.53/hr). Handles: FAQs, simple questions, classifications, short answers. Response time: <50ms (cache) or <500ms (2B model).
$0.53/hr · <500ms
Tier 2 — Gemma 27B AWQ + Spot (normal queries)
Main workhorse. AWQ 4-bit on g5.12xlarge Spot fleet. Handles: complex chat, RAG answers, code generation. With prefix caching + speculative decoding. Response time: 1–3s streamed.
~$1.70/hr Spot · 1–3s
Tier 3 — Agent mode On-Demand (complex tasks)
Async job queue (SQS → worker). Multi-tool agent runs. On-Demand instances (never Spot for long tasks — preemption mid-agent is catastrophic). Response time: 5–30s. Use WebSocket to stream intermediate agent steps to user.
On-Demand · async · 5–30s
